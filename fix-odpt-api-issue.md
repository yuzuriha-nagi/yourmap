# ODPT API問題の修正指示

## 問題
現在のODPT APIは九州地域（JR-Kyushu、Nishitetsu、FukuokaSubway）のデータを提供していません。
APIからは空の配列 `[]` が返されており、これが404エラーの原因の一つとなっています。

## 対応方法（以下のいずれかを選択してください）

### 選択肢1: モックデータの使用（推奨）
現在のコードには既にモックデータ生成機能が実装されているため、環境変数を設定してモックデータを使用する。

**必要な作業:**
```bash
# .env.localファイルに以下を追加
echo "NEXT_PUBLIC_USE_MOCK_DATA=true" >> .env.local
```

### 選択肢2: 東京圏のデータに変更
九州のデータではなく、ODPT APIが提供する東京圏のデータ（JR東日本、東京メトロ等）を使用するようコードを変更する。

**対象オペレーター例:**
- `odpt.Operator:JR-East`
- `odpt.Operator:TokyoMetro`
- `odpt.Operator:Toei`

### 選択肢3: 代替API検索
九州地域の公共交通データを提供する別のAPIを探して統合する。

**候補:**
- 各交通機関の公式API
- オープンデータポータル
- 交通系ICカードデータ

## 推奨する対応
**選択肢1のモックデータ使用**を推奨します。理由：
1. 既存コードの修正が最小限
2. デモ用として十分な機能
3. 将来的に実際のAPIが見つかった際の移行が容易

どの選択肢で進めるかお知らせください。
